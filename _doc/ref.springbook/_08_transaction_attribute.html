Transaction 속성

Transaction Propagation 트랜잭션 전파

PROPAGATION_REQUIRED
진행중인 트랜잭션이 없으면 새로 시작, 이미 시작된 트랜잭션이 있으면 참여 (DefaultTransactionDefinition 의 트랜잭션 전파 속성)
PROPAGATION_REQUIRES_NEW
이미 진행중인 트랜잭션의 존재여부와 관계없이 항상 독립적인 트랜잭션 생성
PROPAGATION_NOT_SUPPORTED
트랜잭션을 시작하지 않음. 특정 pointcut 에 트랜잭션을 제외하고자 할때 유용
기타

ISOLATION LEVEL 격리수준

ISOLRATION_DEFAULT
DataSource 에 설정되어있는 디폴트 격리수준을 따름 (DefaultTransactionDefinition 의 격리수준)
기타

TIMEOUT 제한시간

트랜잭션을 수행하는 제한시간. DefaultTransactionDefinition 의 기본설정은 제한시간이 없는것이다.
트랜잭션을 직접 생성할 수 있는 PROPAGATION_REQUIRED 나 PROPAGATION_REQUIRES_NEW 트랜잭션 전파 속성과 함께 지정되어야 의미가 있다.

READ ONLY 읽기전용

읽기전용으로 설정해두면 트랜잭션 내에서 데이터를 조작할 수 없게 할수 있다. 데이터 액세스 기술에 따라 성능향상을 꾀할 수 있다.
